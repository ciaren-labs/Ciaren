# FlowFrame Roadmap

Our vision for where FlowFrame is heading. Join us on this journey!

## Current Version: v0.1 (MVP)

### ✅ Completed

- [x] Visual flow editor
- [x] 25+ transformation nodes
- [x] Live data preview
- [x] Python code export
- [x] Flow save/load
- [x] Run history
- [x] SQLite, PostgreSQL, MySQL support
- [x] Comprehensive documentation
- [x] GitHub Actions CI/CD
- [x] Open-source with MIT license

## v0.2 (Learning & Community)

**Target:** Q3 2026

### Documentation & Learning

- [ ] Video tutorials (YouTube)
- [ ] Interactive flow examples (embedded editor)
- [ ] Jupyter notebook integration
- [ ] Blog posts on data cleaning
- [ ] Community gallery (user-submitted flows)

### Core Features

- [ ] Additional transformations:
  - [ ] Text operations (trim, split, concat)
  - [ ] Date operations (extract, format)
  - [ ] Math operations (round, abs, power)
  - [ ] Window functions (lag, lead, rank)
- [ ] Improved error messages
- [ ] Data validation & quality checks
- [ ] Flow comments & documentation within editor

### Backend Improvements

- [ ] Async execution (parallel transforms)
- [ ] Better performance metrics
- [ ] Execution logs with timing
- [ ] Flow versioning

## v0.3 (Production Ready)

**Target:** Q4 2026

### Deployment & Scheduling

- [ ] Simple scheduling (cron-like)
- [ ] Slack notifications
- [ ] Email reports
- [ ] Command-line interface (`flowframe run flow.json`)
- [ ] Docker deployment
- [ ] Kubernetes support (optional)

### Advanced Features

- [ ] Custom transformation nodes (user-defined)
- [ ] Conditional branching (if/then/else)
- [ ] Loop iterations
- [ ] Variable management
- [ ] Flow templates

### Integrations

- [ ] Database connectors:
  - [ ] PostgreSQL input/output
  - [ ] MySQL input/output
  - [ ] SQLite input/output
- [ ] Cloud storage:
  - [ ] S3 support
  - [ ] GCS support
  - [ ] Azure Blob support
- [ ] API connectors:
  - [ ] REST API input
  - [ ] GraphQL input

## Future (v2.0+)

Beyond v1.0, we're exploring:

- **Distributed execution** — Dask/Spark backends
- **Real-time streaming** — Kafka, Flink integration
- **ML integration** — Scikit-learn, XGBoost nodes
- **Mobile app** — iOS/Android companion app
- **AI-powered generation** — "Describe your flow, I'll build it"
- **Internationalization** — Multiple languages
- **SaaS option** — Managed hosting (optional)

## How to Request Features

1. **[Check existing discussions](https://github.com/rodrigo-arenas/FlowFrame/discussions)** — avoid duplicates
2. **[Open a discussion](https://github.com/rodrigo-arenas/FlowFrame/discussions)** — describe your use case
3. **Upvote features** you want to see
4. **[Contribute code!](/CONTRIBUTING.md)** — implement features yourself

## Contributing to the Roadmap

We prioritize features based on:

1. **Community demand** — upvotes and discussions
2. **Real-world use cases** — what problems you're solving
3. **Technical feasibility** — complexity vs. impact
4. **Maintainability** — can we support it long-term?

Your feedback shapes our roadmap. [Join the discussion! →](https://github.com/rodrigo-arenas/FlowFrame/discussions)
