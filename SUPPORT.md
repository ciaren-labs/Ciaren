# Support

Ciaren is alpha software maintained by a small open-source team. Please use
the right channel so questions, bugs, and security reports do not get mixed
together.

## Questions and Troubleshooting

Use GitHub Discussions for:

- Setup help
- Usage questions
- "How should I model this flow?" questions
- Early feature ideas
- General troubleshooting

Start here: <https://github.com/ciaren-labs/Ciaren/discussions>

## Bugs

Use GitHub Issues for reproducible bugs, regressions, crashes, or incorrect
behavior:

<https://github.com/ciaren-labs/Ciaren/issues/new/choose>

Before opening an issue, search existing issues and PRs. Include reproduction
steps, environment details, logs, and anonymized sample data when possible.

## Feature Requests

Open an issue for concrete feature proposals. Use Discussions first for broad
brainstorming or unclear product direction.

## Connectors and Integrations

Ciaren keeps the open core intentionally lightweight. Maintainers will not add
every database, SaaS API, internal service, or proprietary storage system as a
built-in connector.

For new integrations, use the Plugin SDK and discuss the design in GitHub
Discussions. Open an issue only when there is a reproducible bug in an existing
connector, a documentation gap, or an SDK/API limitation that prevents a plugin
from being built.

## Security

Do not open public issues for vulnerabilities. Follow the private reporting
process in [SECURITY.md](SECURITY.md).

## Maintenance Expectations

Ciaren does not currently offer commercial support or guaranteed response
times. Maintainers prioritize security reports, regressions, and issues with
clear reproductions.
